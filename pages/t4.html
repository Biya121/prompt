APE 기법
Automatic Prompt Engineer — AI에게 프롬프트를 직접 만들어 달라고 요청하는 기법입니다. 내가 원하는 작업과 결과물을 설명하면, AI가 여러 후보 프롬프트를 생성하고 스스로 평가한 뒤 최적의 프롬프트를 골라줍니다. 자주 반복하는 작업일수록, 좋은 프롬프트 하나를 만들어 두는 것이 훨씬 효율적입니다.
APE 4단계 프로세스
1
후보 생성 서로 다른 전략을 사용하는 프롬프트 후보 10개를 만듭니다.
2
자체 평가 명확성·완전성·실행 가능성·품질 유도력 4가지 기준으로 각 후보를 채점합니다.
3
교차 최적화 상위 3개를 결합해 각각의 강점만 모은 최적 프롬프트 1개를 만듭니다.
4
최적 프롬프트 출력 완성된 프롬프트 전문, 선택 이유, 사용 시 주의사항을 함께 제공합니다.
10가지 프롬프트 전략 유형

범용 APE 마스터 프롬프트
어떤 작업이든 아래 마스터 프롬프트에 작업 내용만 채워 넣으면 AI가 최적 프롬프트를 자동으로 설계합니다.
#역할 당신은 프롬프트 엔지니어링 전문가입니다. 주어진 작업에 대해 최적의 프롬프트를 자동으로 생성 / 평가 / 개선합니다. ##STEP 1 : 후보 프롬프트 10개 생성 서로 다른 전략을 사용해 후보 10개를 만드세요. (역할 부여형 / 단계별 지시형 / 예시 포함형 / 제약 명시형 / 목적 중심형 / 독자 설정형 / 형식 지정형 / 품질 기준형 / 역방향형 / 메타 지시형) ##STEP 2 : 후보 자체 평가 각 후보에 대해 4가지 기준으로 점수를 매기세요. - 명확성 : 지시가 모호하지 않은가? - 완전성 : 필요한 정보가 모두 있는가? - 실행 가능성 : AI가 바로 실행할 수 있는가? - 품질 유도력 : 높은 품질의 결과를 이끌어 내는가? ##STEP 3 : 상위 3개 교차 최적화 상위 3개의 강점을 결합해 최적 프롬프트 1개를 만드세요. ##STEP 4 : 최적 프롬프트 출력 - 최적 프롬프트 전문 - 왜 이게 최선인지 설명 - 사용 시 주의사항 --- 작업 : {자동화하고 싶은 작업을 설명} 원하는 결과물 : {어떤 결과물이 좋은 결과물인지} 피해야 할 것 : {어떤 결과물이 나쁜 결과물인지}
응용 · 코딩 전용 APE
코딩 작업에 특화된 APE 버전입니다. 실행 가능성·완전성·가독성·확장성 4가지 기준을 코딩에 맞게 조정합니다.
코딩 작업을 위한 최적 프롬프트를 생성하세요. 작업 : {무엇을 만들 것인가} 언어 / 프레임워크 : {Python / JavaScript / 등} 제약 : {성능 요구, 라이브러리 제한 등} 경험 수준 : {초보 / 중급 / 고급} APE 평가 기준 : 1. 실행 가능성(30점) : 코드가 바로 실행되는가? 2. 완전성(25점) : 엣지 케이스, 에러 처리 포함하는가? 3. 가독성(25점) : 주석과 변수명이 이해하기 쉬운가? 4. 확장성(20점) : 나중에 수정하기 쉬운 구조인가? 4가지 기준 합산 80점 이상의 프롬프트만 최종 후보로 선택하세요.
✅
APE가 가장 빛나는 순간
같은 유형의 작업을 반복적으로 해야 할 때입니다. 예를 들어 매주 바이어 이메일을 써야 한다면, 한 번의 APE로 최적 이메일 프롬프트를 만들어 두고 계속 재사용할 수 있습니다.
