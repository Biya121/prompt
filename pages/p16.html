아웃라인 확장 패턴
복잡한 주제를 한 번에 상세하게 요청하면 방향이 어긋나기 쉽습니다. 이 패턴은 큰 구조(아웃라인)를 먼저 확인한 뒤, 원하는 파트만 단계적으로 깊이 확장하는 방식입니다. 교육과정, 보고서, 사업 계획서처럼 구조가 복잡한 결과물에 특히 효과적입니다.
3단계 확장 구조
1
대주제 아웃라인 최상위 구조와 각 항목의 제목·한 줄 설명만 작성. 세부 내용은 아직 X.
2
선택 파트 확장 원하는 특정 파트만 골라 상세 아웃라인으로 확장. 학습 목표·실습 내용·시간 포함.
3
최소 단위 완성 특정 하위 항목을 완성본(스크립트·코드·문서)으로 작성.
1단계 · 대주제 아웃라인 요청
전체 구조의 큰 그림만 먼저 봅니다. 세부 내용 없이 항목 제목과 한 줄 설명만 요청해 방향이 맞는지 확인합니다.
아래 주제의 최상위 구조만 작성하세요. 각 항목의 제목과 한 줄 설명만 포함하세요. 세부 내용은 아직 작성하지 마세요. 주제 : "무역 데이터 자동화 심화 과정" (총 40시간)
2단계 · 선택한 파트 확장
1단계에서 확인한 구조 중 원하는 파트를 골라 상세하게 확장합니다. 학습 목표, 실습 내용, 예상 소요 시간까지 포함시킵니다.
다섯 번째 주의 상세 아웃라인을 작성하세요. 각 수업에 아래 내용을 포함하세요. - 학습 목표 - 실습 내용 - 예상 소요 시간
3단계 · 최소 단위 완성본 작성
아웃라인의 특정 항목 하나를 실제 사용 가능한 완성본으로 작성합니다. 강의 스크립트, 실습 코드, 체크리스트 등 바로 쓸 수 있는 결과물을 얻습니다.
3일차 수업의 강의 스크립트와 실습 코드를 완성본으로 작성하세요.
응용 · 비중과 우선순위 포함 아웃라인
각 파트의 전체 대비 비중(%)과 우선순위를 함께 표시하면, 시간 배분과 중요도 파악이 한눈에 됩니다.
아래 교육과정의 아웃라인을 작성하되 각 파트의 전체 대비 비중(%)과 우선순위를 함께 표시하세요. [조건] - 실습 코드가 이론 설명보다 비중 높아야 함 - 수강생 수준 : Python 기초 수료자 - 총 교육 시간 : 40시간 주제 : "무역 데이터 자동화 심화 과정"

핵심 원칙
한 번에 모든 것을 요청하지 않습니다. 먼저 큰 그림을 확인하고, 방향이 맞으면 그때 세부를 요청하세요. 이 방식이 결과물의 방향 오류를 가장 효과적으로 방지합니다.
