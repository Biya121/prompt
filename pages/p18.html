무한 생성 패턴
변수만 바꾸면 같은 구조의 결과물을 무한히 만들어 낼 수 있는 패턴입니다. 매번 새로 프롬프트를 작성하는 대신, 틀(템플릿)을 한 번 설계해 두고 변수만 교체하는 방식입니다. 이메일, 보고서, 제안서처럼 반복 생성이 많은 업무에서 큰 효율을 발휘합니다.
핵심 구조
🔑
템플릿 제목 → 변수 정의 → 생성 규칙 → 변수 값만 바꿔 무한 생성
예시 · 무역 이메일 생성기
수신자, 목적, 톤 3가지 변수를 조합해 상황에 맞는 무역 이메일을 즉시 생성합니다. 템플릿을 한 번 설정해 두면 변수만 바꿔 다양한 상황의 이메일을 빠르게 만들 수 있습니다.
무역 이메일 생성기 "[수신자 유형]에게 [목적]을 전달하는 영문 이메일" 변수 : {수신자} : 신규 바이어 / 기존 거래처 / 포워더 / 관세사 {목적} : 첫 연락 / 견적 요청 / 납기 연장 / 클레임 대응 {톤} : 공식 / 친근 / 긴급 생성 규칙 : - 3단락 구조 - 제목 포함 - 150단어 이내 요청 예시 : 수신자 = 신규 바이어 목적 = 첫 연락 톤 = 공식
변수 조합 예시
수신자목적톤결과물 성격
신규 바이어첫 연락공식정중한 소개 이메일
기존 거래처납기 연장친근양해 구하는 부드러운 이메일
포워더견적 요청긴급빠른 답변을 촉구하는 이메일
관세사클레임 대응공식법적 표현이 포함된 공식 서한
응용 · 나만의 생성기 만들기
같은 구조로 자신의 업무에 맞는 생성기를 직접 설계할 수 있습니다. 변수의 종류와 생성 규칙을 바꾸면 어떤 반복 업무에도 적용됩니다.
[생성기 이름] : {결과물 이름} 생성기 변수 : {변수1} : 선택지A / 선택지B / 선택지C {변수2} : 선택지A / 선택지B / 선택지C {변수3} : 선택지A / 선택지B 생성 규칙 : - {형식 조건} - {길이 조건} - {포함해야 할 요소} 요청 : 변수1 = ? 변수2 = ? 변수3 = ?
💡
설계 팁
변수는 3~4개가 적당합니다. 너무 많으면 조합이 복잡해져 결과물의 일관성이 떨어질 수 있습니다. 가장 자주 바뀌는 요소만 변수로 만들고, 나머지는 생성 규칙으로 고정하세요.
